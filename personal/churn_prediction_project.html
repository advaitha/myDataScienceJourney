Check-off list for Data Science Projects

Project Understanding

  • Understanding of the business for whom we are doing this project. What is the business model? How revenue is generated by the business?

    • SVOD and AVOD model

  • Why this project is required? How this project will contribute to the business?

    • Subscribers who were taking subscriptions were dropping off.
    • If we are able to retain these subscribers then it will help increase the company’s revenue
    • More revenue will help create more interesting content. This will increase choice of options on the platform. More choice, customers are more happy. Once enough quality content is available, marketing campaigns can be increased to acquire customers
    • In this way churn was tied to a lot of things in the organization - content strategy, marketing, customer success etc.

  • what is the business metric which we are trying to optimize? Which metrics will be adversely affected by trying to optimize the metric we are interested? What steps can be taken to overcome the imbalance?

    • Increase in watch time
    • Increase in videos watched
    • Customers quality of life - Spending lot of time watching videos. Healthy interventions like “You are watching videos from a long time, would you like to take a break?” will help improve the customer satisfaction

  • Does it require data science solution? Can we solve the business problem with data science or with data analysis and hypothesis testing?

    • Talk to domain experts and understand their view points.
    • we can start with simple analytics and see if the analysis match with the view of domain experts
    • If there is difference between two views think why this may be happening
    • Share the findings, frame a hypothesis and test it with data
    • Use conclusions from hypothesis testing to improve business decisions
    • For example, is there any behaviour difference between users who are planning to churn and not churn in the last few weeks of subscription?

  • What is the data generation process?

    • Defining the events, code for capturing the data, storing the data etc.

  • Is the data getting captured required for the project? Yes

  • What is the volume, variety and speed of the data generated? - 5% of the users were from SVOD. Data for subscribers was in GBs (Around 10). Structured data. A lot of new customers were getting acquired. A scalable solution was required to handle the speed of the data.

  • What about data quality? - A few challenges were there, nothing serious which will jeopardize the project

  • Is this a batch requirement or real-time requirement? - Batch

  • What are the infrastructure needs for the project? - Storage requirements, Compute requirements

  • What help would be required from cross-functional teams? - DE team to prepare data pipelines, access to cloud and provisiong of resources, devops for deploying the model etc.

  • What are the trade-offs involved in the project? Metric selected, doing it for a region with low risk profile, may be we are losing oppurtunity by selecting such a region, actions we can take once we predict a user is going to churn - How are we going to stop churn? Does giving discounts solve the purpose? Will users game the system? what about giving a discount to a customer who the model predicted as churn but in actual he would have retained? Usually to improve subscription numbers, discounts are provided towards the end of the month. Do users take adverse advantage of this discount?

  • What challenges can we foresee in project implementations? Any projects with similar nature and what are the lessons learnt? - Same as above, availability of resources, including getting bandwidth of other teams, tech capability required to deploy, evaluate and monitor the systems - For example the capacity to do AB testing etc.

  • Can we start with something simple before going in for complexity? - Yes, a simple random generator which will output either churn / not-churn. Use this output and see how we can use the same for business decisions and execution. We din’t do this, if given a second oppurtunity will start with this one and move ahead from there.

  • Are there any ethic issues here? what can be done to make sure that our society is not adversely impacted by this project? - Yes, the metric selected may be tweaked to improve human condition. Analyze if there is any difference in the discounts given to any gender, age or race etc?

Data understanding

  • See if column names are proper and needs to be fixed

  • Understand the each and every column in the data and what they mean - Is there a data dictionary existing for the data? - We preapred a data dictionary for the project

  • What is the granularity of the data? which column should be treated as primary key?

    • Each row of the data represented a single subscriber.
    • 25 days of data for each user is aggregated and used as features
    • For example if a user got subscription on 05th of November till 04th of December, then the data for aggregation is considered from 05th of November till 30th of November
    • Features used for building the model:-
      • Main categories are:-
        • User behaviour metrics / demographic metrics
        • Quality of video metrics
        • Content metadata
        • Customer success metrics
      • Number of times logged on to the platform
      • Videos liked / Skipped
      • Age and gender of the user
      • Location of the user
      • When this video was watched
      • Video views
        • Movies
        • TV Shows
        • Genre count of the videos
      • Watch time
      • Language watched - total of videos watched in various languages
      • Video quality metrics
        • Number of times video buffered
        • Device on which video was watched
        • Bit rate
        • Time it took to load the first frame of the video
        • Number of playback errors encountered
      • Number of times reached out to customer service

  • How was the data generated? Understand the data collection process? Does the way data generated and collected impacts the analysis in any way?

  • where is the data stored? How is to be accessed?

  • Is there any PII information which should be masked?

  • Any variables which will impact ethics like age, race etc? - Age and gender was used. We should have avoided the same.

Data Exploration

  • Check the datatypes
  • Change the datatypes if required
  • Explore and visualize the data
    • See if there are duplicated records
      • Duplicated records on row level
      • Duplicated records on primary key level
    • Check missing values and their percentage in a column
    • Numeric columns
      • Summarize the numerical values and see the distribution
      • Visualize the distribution if required
      • Is there high correlation between numeric values in the data? Visualize the correlation
      • Are there outliers in the data? What is the percentage of outliers? what to do with outliers?
      • Is scaling required on numeric columns - Which strategy on which column?
      • Missing value Imputation - which strategy for which column?
    • Categorical values
      • Any data cleaning required?
      • Capital letters to be converted to lowercase?
      • Are there any spelling mistakes in the values captured?
      • What is the cardinality of the data? And what are the value counts of each category?
      • what kind of encoding will be required for each categorical column?
      • Missing value Imputation - which strategy for which column?
    • Date features
      • Are all date features in the same timezone?
      • What time zones are being used?
      • Is time zone conversion required?
      • What features should be extracted from the date column?
      • Missing value imputation - which strategy for which column?
    • Target column
      • Regression - How is the data distributed? Any transformation required?
      • Classification - Is the dataset balanced? Any upsampling or downsampling required to balance the data? 85% not churn and 15% churn - we undersampled the training data for not churn to balance the data. As the amount of data was high, we did not oversample the data.

Data Cleaning, Preprocessing and Feature Engineering

  • Remove the duplicated records - Row level and primary key level
  • Decide on data split strategy
    • If the data is huge, split the data into train, validation and test
    • If the data is less then go for cross-validation
  • All the below needs to be done on all data splits -
    • Date
      • Extract the required features from date column and check the dtypes of extracted features
    • Drop columns
      • Date column from which features are extracted
      • Highly correlated numeric columns (based on domain understanding)
      • Columns with very high missing values based on threshold
      • Columns not making business sense
    • If required separate out the numeric and categorical columns for ease of processing
    • Numeric columns
      • Outlier treatment if required
      • Treat missing values
      • Scaling - Appropriate strategy
    • Categorical columns
      • Clean the categorical values as required
      • Missing value treatment
      • Decide on the encoding depending on cardinality and value counts
    • Target column
      • Regression - Transform the target column if required
      • Classification
        • Balance the dataset if required (only train - Crosscheck)
        • Encode the target column if required
    • Combine all numeric and categorical columns into a single dataset
    • Can we construct any intelligent features from the data?
    • Analyze carefully if there is any data leakage

Modelling

  • Finalize the metric which needs to be optimized - For a baseline model, we analyzed different ML metrics to see which is positively correlated with business metrics. Selected the ML metric based on analysis. Recall was showing most promise hence selected recall.
  • Does the Data science metric selected impact the business metric positively? Is there a way to test this hypothesis? - Analysis was done to conclude this.
  • Select a few models depending on the requirements:-
    • Batch or real time requirement
    • Number of requests expected & Latency
    • Volume, variety and speed of data
    • Size of the model required
    • Device on which this needs to be deployed
    • Infrastructure requirements
    • Any other requirements
  • Spot check a few models on the data without doing hyperparameter tuning - We used a few models like Decision Tree, Random Forest, SVM, KNN to check which one was performing better. We din’t decide to go for KNN even though it was showing promise because it is a lazy learner and it will need complete data stored in memory for predictions. As we were growing fast, we did not for KNN. SVM also can’t be run parallely, so avoided that.Keeping in view the growth in future, we went ahead with tree based model. Decision tree results were not stable hence we went ahead with random forest. (With a simple change in the feature there was a drastic change in the output of the model).
  • Identify the models which are performing well on the data.
  • Tune the hyperparameters for the selected models and test it on validation data - No of trees to build, max depth of the tree, max number of nodes, number of instances required in each group for a split etc.
  • Analyze the model performance on the test data - Accuracy - 90%, Recall - 75%, Precision - 70%, we checked the predictions at different thresholds and selected the threshold which was giving us good recall.
  • Experiment with the models till you are satisfied
  • Finalize the model based on evaluation of results - Based on this recall the model was finalized for deployment.
  • Check the performance of the model on different slices or categories of the data - We did analyze for various segment groups how this model was performing. For some age groups and regions it was not performing below the overall average.

Important considerations

  • Version control the data used for the project
  • Keep the code under version control at all times
  • Logging the process
  • Capture the values of parameters for every experiment - Use a framework like mlflow
  • Create a separate environment for the project
  • Keep track of the packages and their versions used for the project
  • Documentation
    • Every project should have a proper Readme file along with instructions for recreating the environment and executing the code
    • Write docstrings for every class and function
    • Always keep the documenation upto date

  • Check the impact on business metrics?
  • Monitor how the models are performing in production
  • Monitor for data and concept drift
  • what about continual learning for the models? Can we train the model continously on recently generated data?
  • If continual learning is not possible, what is the optimum time period for model retraining?
  • What improvements can be done on all the steps involved?
  • How can we repeat the steps in an optimized way?
